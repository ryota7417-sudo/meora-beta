MEORA - アプリ概要
==================

気軽な話し相手キャラ x クリエイターマーケット のチャットアプリ。
10〜30代女性をメインターゲットに、自分だけのAIキャラクターと会話できる。


コンセプト
----------
ChatGPTを「チャッピー」と呼んで話し相手に使っている層に向けて、
キャラクターIPを持つクリエイターが参入できるプラットフォームを目指す。
専門AI（ライター・デザイナー等）から事業ピボットし、
「気軽な話し相手」に特化した設計に転換した（2026-06-15）。


デザイン世界観
--------------
テーマ: レトロ x AI
- フォント: Nosutaru-dotMPlusH-10-Regular（ドット系レトロフォント）をアプリ全体に統一
- UI: 方眼紙背景 x 白カード x 黒太枠 x 影（boxShadow: 4px 4px 0 #111）
- アイコン: 黒線モノクロSVG
- 絵文字（色付きピクトグラム）は使用禁止。矢印等の記号は可
- 配色: 背景 #f7f5f0 / 文字 #111 / アクセント #e8568a

画面構成
--------
/                     スプラッシュ → /onboarding へリダイレクト
/onboarding           初回セットアップ（Googleログイン → アバター作成）
/dashboard            ホーム画面。上部にキャラが歩くエリア、下部にトーク一覧
/chat/[id]            キャラとの1対1チャット画面
/character/new        自作キャラ作成（名前・性格・写真・スプライト設定）
/character/edit/[id]  既存キャラの編集
/market               クリエイターマーケット（キャラ・お食事・スキンの3タブ）
/market/character/[id] マーケットキャラの詳細・入手
/market/shop/[id]     クリエイターのお店ページ
/settings             設定（アカウント・購入履歴・キャラ管理・ログアウト）


APIエンドポイント
-----------------
POST /api/chat        チャットAPI。GPT-5.4 miniにリクエストを中継
GET  /auth/callback   Supabase OAuth コールバック（Googleログイン後のリダイレクト先）


主要ライブラリ（lib/）
----------------------
store.ts       アプリ状態の管理。キャラ・チャット履歴をlocalStorageに読み書き。
               HP（満腹度）システム、毎日AM5:00のおにぎり自動支給も含む。
supabase.ts    Supabaseクライアントの初期化（ブラウザ用）
auth.ts        ログイン直後のセッション競合を解決するヘルパー。
               「肯定は速攻採用 / 否定は猶予と再確認の後に採用」方式。
crisis.ts      危機ワード検知。ユーザー発話に自殺・自傷等のキーワードが含まれた場合、
               モデル出力とは別に、正規の相談窓口（いのちの電話等）を確定文として付加。
               番号はアプリ側でハードコードし、モデルには生成させない。
i18n.ts        多言語対応（日本語 / 英語 / 中国語繁体字）
market-data.ts マーケットのサンプルデータ（キャラ・クリエイター・アイテム）。
               バックエンド未実装のため静的データで構成。
inherit-persona.ts  キャラの性格引き継ぎ機能


コンポーネント（components/）
-----------------------------
CharacterYard.tsx   ホーム上部のキャラ歩行エリア。requestAnimationFrameで
                    位置をref+直接style操作で更新し、Reactの再レンダーを最小化。
                    常時歩行し、たまに短い休憩を挟む。スプライトサイズ 112px。
BottomNav.tsx       下部ナビゲーションバー（ホーム・マーケット・設定）
CharacterSvg.tsx    キャラアバター表示（写真 or イニシャル）
ComingSoonToast.tsx 「準備中」トースト表示
Icons.tsx           モノクロSVGアイコン集


安全機能
--------
1. システムプロンプトの安全ブロック
   - キャラ脱却防止（「AIでしょ?」への耐性）
   - 未成年保護（危険な状況での適切な対応指示）
   - メンタルヘルス対応（共感優先、窓口案内は一般名称のみ）

2. アプリ側ガードレール（crisis.ts）
   - 危機ワード検知でモデル出力に関わらず正規窓口の電話番号を確定表示
   - 番号・窓口名はハードコードし、モデルの創作リスクを排除

3. HP（満腹度）システム
   - 1回の会話で5HP消費。HP 0で会話不可
   - 毎日AM5:00におにぎり1個（30HP）自動支給
   - 課金アイテムでHP回復（決済は未実装）


データ保存
----------
現状は全てlocalStorage:
- meora-state         アプリ状態（キャラ一覧・HP・オンボーディング完了フラグ等）
- meora-chat-{id}     キャラごとのチャット履歴
- meora-onboarding-step  オンボーディングの進捗

Supabaseは認証のみ接続済み。DBテーブル（characters / chat_messages / purchases）と
Supabase Storageへの移行は次フェーズで予定。


収益モデル（設計済み・未実装）
------------------------------
- クリエイターマーケット: クリエイターが自分のIPキャラを登録・販売
- 分配: クリエイター 80% / MEORA 20%（ストア手数料控除後）
- サブスク: 月額金額相当のアイテムを毎月付与する前払い型
- 投げ銭: 分配 85/15 で検討中
- アイテム分類:
  - 消費型（おにぎり等）: 推論コスト発生。付与上限あり
  - コスメ系（スキン等）: 推論コストなし。クリエイター自由設定
